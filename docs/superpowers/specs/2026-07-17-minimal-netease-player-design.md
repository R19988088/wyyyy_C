# 极简网易云音乐播放器设计规格

## 1. 目标与范围

首版是一个 Android 优先、可继续扩展到 Windows、macOS 和 Linux 的极简网易云音乐播放器。应用以 Rust 承担网易云协议、账号数据、状态持久化和播放地址解析，以 Tauri 2 WebView 承担界面与音频输出。

首版只包含：手机号验证码登录、个人专辑、个人歌单、个人订阅播客、集合详情、在线播放、播放状态恢复、设置入口。首版不包含搜索、下载、歌词、均衡器、调试页面和多平台音源。

## 2. 技术方案

- 容器：Tauri 2，优先生成 Android Release APK，并保留桌面目标所需的共享代码边界。
- 前端：Vite + 原生 TypeScript、HTML、CSS，不引入组件框架。
- Rust：`reqwest`、`serde`、`tokio` 及必要的网易云 WEAPI/EAPI 加密依赖。
- 音频：WebView 的单一 `HTMLAudioElement`。Rust 只返回经过账号验证的播放地址，前端统一维护播放、暂停、进度和错误事件。
- 存储：Rust 侧保存登录 Cookie 和账号资料；前端持久化界面选择与每个集合的播放位置。持久化数据按账号 ID 隔离。

前端不直接调用网易云接口，也不保存手机号验证码。Rust 命令是账号、媒体库和播放地址的唯一数据入口。

## 3. 主界面

主界面使用单页、无底部导航结构：

1. 顶部左侧为 `专辑 / 歌单 / 播客` 三个文本标签，当前标签使用实色强调。
2. 顶部右侧为齿轮图标按钮，点击后打开设置抽屉。
3. 中部是 Cover Flow 封面区域。中心封面完整正面显示并带大范围柔和投影；相邻封面缩小、降低亮度并沿 Y 轴透视倾斜，露出后续内容。
4. 底部显示当前播放曲目名称、作者、进度和圆形播放/暂停按钮。

封面、播放器和列表都使用稳定尺寸，任何动态文本不得推动主要控件。窄屏时曲目名称单行截断，封面区域保持正方形中心封面。

## 4. 封面交互

- 左右滑动中心区域，或点击两侧封面，只改变当前浏览焦点。
- 浏览焦点变化时，正在播放的音频、播放队列和进度保持不变。
- 侧面封面第一次点击后移动到中央；只有再次点击已经居中的封面才确认切换播放集合。
- 确认切换后，播放器恢复该集合上次播放的曲目和曲内进度；该集合无历史记录时从第一首开始。
- 上下滑动封面切换到当前浏览集合的曲目列表；在列表顶端继续执行同方向的有效滑动时返回封面。
- 上下切换使用位移动画，不与列表本身的正常滚动手势冲突。
- 切换顶部类别时只改变浏览数据，不自动切换正在播放的集合。

所有动画尊重系统减少动态效果设置。Cover Flow 的透视、缩放和阴影只表达空间层级，不触发媒体状态变化。

## 5. 曲目列表

列表展示曲目序号、名称、作者和时长。当前播放曲目使用网易云红色强调，不额外添加装饰卡片。

点击任意曲目立即切换到该曲目并开始播放，同时更新该集合的恢复位置。列表加载中显示骨架占位；空集合显示明确空状态；接口失败显示错误和重试按钮。

## 6. 登录与账号数据

设置抽屉首版只有登录状态与登录操作：

1. 未登录时输入国家/地区代码和手机号，点击发送验证码。
2. 输入验证码后提交登录。
3. Rust 调用网易云短信验证码接口和手机号验证码登录接口，提取并持久化 `MUSIC_U`、`__csrf`、`os`、`appver` 等会话 Cookie。
4. 登录成功后获取当前账号 ID，并并行加载个人创建/收藏歌单、收藏专辑和订阅播客。
5. 已登录时显示账号昵称和退出登录按钮。退出会清理 Cookie 与内存数据，但保留该账号独立的播放位置，供再次登录恢复。

验证码发送按钮包含倒计时和请求中禁用状态。接口错误直接显示网易云返回的可读错误，不伪装成登录成功。

## 7. 播放与恢复

每个专辑、歌单和播客使用 `类型 + 集合 ID` 作为持久化键，记录：曲目 ID、曲目索引、播放进度、更新时间。

应用还记录最近一次活动集合、曲目、进度以及退出前的播放/暂停状态。启动流程是：

1. 恢复登录 Cookie 并校验账号。
2. 恢复最近活动集合和曲目元数据。
3. 重新解析可用播放地址。
4. 将音频定位到保存进度。
5. 退出前为播放状态时恢复播放，退出前为暂停状态时保持暂停。

进度每 5 秒以及暂停、切歌、进入后台、退出时保存。播放地址不持久化，启动时始终重新解析，避免过期地址导致恢复失败。恢复失败时保留集合与曲目选择并显示重试，不静默跳到其他歌曲。

## 8. 液态玻璃扩展边界

首版 Cover Flow 使用 CSS 透视、阴影和过渡。封面舞台被封装为独立渲染组件，输入只有封面纹理、位置、选中状态和手势状态。

后续真实折射液态玻璃可以把该组件替换为 WebGL 2 渲染器：将背景和封面上传为纹理，通过 fragment shader 进行法线扰动、背景采样、边缘高光和轻微色散。Rust 数据接口、播放状态机和集合恢复逻辑不需要修改。低性能设备或 WebGL 不可用时回退到首版 CSS 效果。

## 9. Android 发布与固定签名

- 本地不执行 Android 编译，所有 APK 构建只在 GitHub Actions 中进行。
- 工作流只运行 Release 构建，不包含 debug APK、调试菜单、测试探针或开发者工具。
- 首次生成一份固定 Android keystore；keystore Base64、别名和密码写入 GitHub Actions Secrets，不提交到仓库。
- 本地保存一份明确命名的 keystore 备份和恢复说明，后续版本始终复用同一签名身份。
- Actions 将签名后的 APK 重命名为 `wyyyy.apk`，直接上传到固定 GitHub Release 标签 `continuous`。
- 不上传 Actions ZIP artifact。下载地址直接指向 Release 中的 `.apk` 资产。

## 10. 验证标准

- Rust 单元测试覆盖网易云加密输入、登录响应 Cookie 提取、媒体库 JSON 解析和播放位置序列化。
- TypeScript 测试覆盖浏览焦点与播放集合分离、二次点击确认切换、集合位置恢复和启动恢复决策。
- Actions 依次运行 Rust 测试、前端测试、Tauri Android Release 构建、APK 签名校验和 Release 上传。
- 使用 Android 模拟器或 Actions 可用的 APK 静态检查确认包名、版本、Release 签名和无 debug 标志。
- 验收时确认 Actions 成功、Release 资产为直接下载的 `.apk`，并记录 APK SHA-256 与签名证书指纹。
