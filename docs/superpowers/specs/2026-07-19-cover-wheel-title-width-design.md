# 封面转轮与标题宽度设计

## 目标

- 封面下方、播放器上方的空白区域作为不可见的圆形转轮输入区。
- 手指沿圆周持续移动时持续翻页，顺时针下一张，逆时针上一张，到达首尾时循环。
- 封面下的标题和副标题使用与封面相同的可用宽度，不再受 `PageView` 窄页宽限制。

## 交互与数据流

转轮区使用指针事件跟踪相对圆心的前后角度，将跨越 `-pi/pi` 边界后的最短角度差换算为圆周切向像素位移。该位移继续传入现有 `CoverScrubSpeedController.update`，保留当前的速度倍率、距离累计和 90 ms 节流逻辑。

速度控制器返回的步数通过模运算映射到收藏数量，从而首尾循环。每次有效步进仍通过现有 `PageController` 动画和 `browseTo` 更新焦点。指针抬起或取消时清空角度状态和速度控制器。

## 布局

不显示圆盘、轮廓或文字。现有 `cover-scrubber` 命中区扩展为蓝圈标注的空白区域，但不覆盖底部播放器。标题区使用 `coverSize` 的固定宽度；标题和副标题仍保持单行省略，避免改变纵向布局。

## 验证

- 单元测试验证圆周角度跨边界换算和方向。
- Widget 测试验证顺时针、逆时针、单次指针手势持续翻页和首尾循环。
- Widget 测试验证标题区宽度接近封面宽度，并保留原有水平快速拖动和竖滑打开列表行为。
