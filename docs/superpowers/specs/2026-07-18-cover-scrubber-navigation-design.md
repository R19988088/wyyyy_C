# 封面快速滚动与返回定位设计

## 顶部导航

- 三个分类在画面中心水平居中，不因右侧按钮宽度偏移。
- 当前分类使用实心背景和对比文字，其他分类保持透明。
- 设置入口改为竖向三点，点击仍进入现有设置页。

## 隐藏快速滚动区

- 位于中心封面标题和创建者下方，高度 56px，无可见背景或标签。
- 只响应水平拖动；轻点和垂直手势不改变封面。
- 参考 `full_20260717_085947` 的 `CapsuleDragSpeedController`：速度分档倍率为 `0/1/3/5/10/20/50`，阈值为 `7/55/90/120/160/200`，步进最小间隔为 90ms。
- Flutter 使用相邻拖动样本的距离/时间估算速度，归一化后进入同一分档表。
- 速度低于有效阈值时，每累积 22px 切换一张；速度和距离同时有效时取步数较大者。
- 向右增加封面索引，向左减少，最终索引限制在当前分类范围内。

## 封面激活

- 中心封面单击不改变播放集合。
- 中心封面双击才激活该集合并开始播放。
- 侧边封面单击只将其移到中心，不开始播放。

## 列表返回定位

- 进入列表时捕获当前 `LibraryKind`、collection ID 和索引，它们是本次列表会话的唯一返回目标。
- 退出列表前按 collection ID 在当前数据中重新定位；如果数据更新使 ID 消失，回退到捕获索引的有效范围。
- 在反向动画开始前重建 `PageController(initialPage: target)` 并同步 `browsedIndex`，不依赖 PageView 离开渲染树后的偏移恢复。
- 反向展开封面、最终中心封面和列表标题必须对应同一 collection ID。

## 验证

- 单元测试覆盖慢速距离步进、快速分档、90ms 节流和范围限制。
- Widget 测试覆盖居中导航、激活背景、三点入口和隐藏拖动区。
- Widget 测试覆盖中心封面单击不播放、双击播放。
- Widget 测试覆盖从非首封面进入列表，退出后目标封面仍精确居中。
