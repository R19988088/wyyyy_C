# 播放器手势与封面列表设计

## 目标

- 大封面区域按手势角度锁定水平或垂直方向，避免一次拖拽连续打开封面列表和歌曲列表。
- 封面列表改为自由上下滚动，不再固定中间选择条或自动居中。
- 滚动每跨过一个封面条目继续触发一次现有震动反馈。
- 双击列表条目后播放对应曲库，并恢复该曲库上次歌曲和播放进度。

## 交互

- 指针移动超过系统触摸阈值后，仅当主方向位移达到副方向的 1.5 倍才锁定方向；模糊角度继续等待，不触发任何列表。
- 水平锁定只切换大封面与当前曲库歌曲列表。
- 垂直锁定只打开封面列表；打开后由普通 `ListView` 接管自由滚动。
- 封面列表没有固定选中位置。双击任意可见条目后切换浏览封面、恢复该曲库播放状态并开始播放，然后关闭封面列表。

## 数据

沿用 Rust 存储中按 `kind:id` 保存的 `SavedPosition`。激活曲库时读取该曲库记录，恢复歌曲索引和秒数；没有记录时从第一首、零进度开始。

## 验证

- Widget 测试覆盖水平优先、垂直优先、模糊斜向手势、自由滚动、滚动震动和双击激活。
- Repository 测试覆盖按曲库恢复歌曲索引与进度。
- 运行静态分析、完整 Flutter 测试和 Rust 测试；Android APK 仅由 GitHub Actions 构建。
