# 正方形封面与全屏列表设计

## 目标

将播放页明确分成封面模式和列表模式，避免封面随剩余高度被拉伸，并在两种模式之间建立连续的展开动画。

## 封面模式

- 顶部显示“专辑 / 歌单 / 播客”和设置按钮。
- Cover Flow 中每张封面的尺寸由可用宽度决定，始终使用 `1:1` 宽高比。
- 封面标题和创建者位于正方形封面下方，不参与封面尺寸计算。
- 保留左右切换时的缩放、透视和投影效果。
- 向上滑动当前内容区进入列表模式。

## 列表模式

- 列表是独立的全屏页面层，不使用封面模式的顶部分类导航。
- 顶部只显示当前专辑、歌单或播客名称。
- 隐藏设置按钮和三种分类切换。
- 歌曲列表占满可用画面，内容可滚动到底部悬浮播放控件下方。
- 底部播放控件在两种模式中都保持悬浮，切换时不重建播放状态。

## 过渡动画

- 采用已确认的 B 方案：当前正方形封面从原位置放大至内容全屏，同时逐渐淡出。
- 列表在封面放大过程的后半段淡入，避免直接闪现。
- 动画使用约 420 毫秒的 `easeOutCubic` 曲线。
- 系统返回手势执行反向动画，恢复到进入列表前的原封面位置。
- 动画期间不切换当前播放集合、曲目或进度。

## 实现边界

- 使用 Flutter 原生布局与动画，不增加新依赖。
- 封面图像继续使用现有持久化缓存。
- 列表进入和退出不重置 `PageController` 或 `browsedIndex`。
- 空集合和曲目加载状态使用同一个全屏列表外壳，保持顶部名称和返回行为一致。

## 验证

- Widget 测试验证中心封面渲染尺寸的宽高相等。
- Widget 测试验证进入列表后分类导航和设置隐藏，仅显示当前集合名称。
- Widget 测试验证列表全屏、底部播放器保留，返回后仍是原封面。
- 运行完整 Flutter 测试和 Dart 静态分析；Android/Rust 构建继续以 GitHub Actions 为准。
