# 专辑切换列表设计

## 目标

- 设置页新增“切换专辑时”类型选择，提供“封面”和“列表”两种模式，默认保持现有“封面”模式并持久化选择。
- 选择“列表”后，按住底部转盘时用纵向专辑列表替代封面切换动画；松手后恢复现有封面播放页。
- 列表仅改变浏览专辑时的视觉反馈，不触发播放、不打开专辑、不改变当前播放内容。

## 交互

设置使用双项分段控件。选择会立即生效并写入现有 `SharedPreferences`。直接拖动大封面仍沿用现有 Cover Flow；“列表”模式只作用于底部转盘，因为列表的出现和消失以转盘按下、松开为边界。

按下转盘后，当前浏览专辑位于列表中央。中央选中带固定不动，使用主题的反色前景和背景；用户转动时，专辑行逐项向上或向下移动。每行左侧显示小封面，右侧显示专辑名和创建者。到达首尾时沿用现有边界回弹，不循环。

列表内容层向下铺满到安全区域底部，不为播放器预留空白。底部专辑行延伸到 `GlassPlayer` 下层，由后绘制的液态玻璃播放器折射显示；播放器控件仍位于列表上方并保持可操作。

松开或取消手势后列表立即退出，恢复为当前浏览专辑的大封面。恢复过程不调用 `activateCentered`，播放器元数据和音频保持原样。

## 实现边界

- 沿用 `PlayerApp` 的设置状态和保存回调模式，不引入新依赖。
- 使用布尔偏好 `listCoverSwitching`，默认 `false`，避免为两个选项增加额外配置抽象。
- 在 `PlayerPage` 中仅当 `listCoverSwitching && scrubberActive` 时渲染列表；复用现有 `_scrubCovers`、`_browseTo`、反馈和边界逻辑。
- 列表以当前 `browsedIndex` 计算各行相对中央的位置并做短动画，不维护第二套浏览索引或滚动状态。

## 验证

- Widget 测试验证设置选择可保存，并且默认仍为“封面”。
- Widget 测试验证列表只在列表模式按住转盘时出现，选中行位于中央且使用反色，小封面位于左侧。
- Widget 测试验证转动后列表上下换项，松手恢复大封面，并且当前播放元数据不变。
- Widget 测试验证列表边界延伸到播放器底部以下，使列表与播放器区域重叠。
