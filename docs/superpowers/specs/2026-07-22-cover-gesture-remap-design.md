# 封面手势重映射设计

## 目标

- 删除设置页的“切换专辑时：封面 / 列表”选项及其持久化状态。
- 取消大封面区域原有的水平 Cover Flow 翻专辑操作。
- 在大封面区域上下拖动时显示现有专辑排序列表，并按纵向移动浏览专辑。
- 大封面页与当前专辑歌曲列表通过任意方向的水平滑动互相切换。

## 手势规则

大封面区域识别纵向拖动后立即显示专辑排序列表。向上拖动映射为下一张专辑，向下拖动映射为上一张专辑；移动速度和步进继续复用 `CoverScrubSpeedController`。松手或取消后恢复选中专辑的大封面，只更新浏览索引，不触发播放。

大封面区域的任意有效水平滑动都打开当前浏览专辑的全屏歌曲列表。全屏歌曲列表上的任意有效水平滑动都返回大封面。歌曲列表的纵向滚动、歌曲点击播放和系统返回手势保持原行为。

底部隐藏转盘继续可用，并固定使用专辑排序列表作为切换反馈。其圆周切换、边界回弹、震动和声音反馈不变。

## 实现边界

- `PageView` 保留为程序化定位和既有封面渲染容器，但使用 `NeverScrollableScrollPhysics` 禁止用户水平翻页。
- 封面内容层统一处理纵向专辑浏览和水平页面切换，手势方向互斥。
- 歌曲列表仅新增水平切换手势，不改变其纵向 `ListView`。
- 删除 `listCoverSwitching` 在 `main.dart`、`settings_page.dart` 和 `player_page.dart` 的全部状态与参数；旧 `SharedPreferences` 键可自然遗留，不再读取或写入。

## 验证

- Widget 测试验证设置页不再显示“切换专辑时”“封面”“列表”。
- Widget 测试验证封面水平拖动不再改变浏览专辑，而是打开歌曲列表；歌曲列表任意水平拖动返回封面。
- Widget 测试验证封面上拖显示专辑排序列表并选择下一张，下拖选择上一张，松手恢复封面且播放元数据不变。
- Widget 测试验证底部转盘仍显示专辑排序列表，纵向歌曲列表仍可滚动。
